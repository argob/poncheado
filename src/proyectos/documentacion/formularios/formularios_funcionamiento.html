Funcionamiento

Los formularios deben funcionar de una determinada manera para ser lo más usable posible.

Errores de validación

La validación sucede (y los mensajes de error aparecen) cuando el usuario aprieta los botones verdes de "Siguiente", "Consultar" o "Enviar".

Valores por defecto de selects

Los selects siempre deben estar vacíos porque si no dan la impresión de que ya están cargados. No se deben usar textos como "Elegí una opción" o demás.

Fuentes de valores de selects

Los valores de los selects deben estar puestos en el html salvo casos particulares (muy largos o que necesiten hacer una consulta a un web service). Si el select va a tardar aunque sea un segundo en mostrar su información, debe aparecer un spinner de carga para indicar eso.

Scroll hacia arriba

Si un formulario de varios pasos está implementado usando una única página HTML, al apretar "Siguiente" la página debe hacer scroll hasta el título del formulario porque si no lo va a tener que hacer el usuario para poder continuar.

Esto no pasa cuando los pasos de carga o las respuestas se manejan en varias páginas HTML.

Guardar datos en el navegador

Cuando el usuario termina de escribir sobre un campo está en foco. Cuando se va el foco de ese campo, lo que escribió el usuario queda guardado usando local storage del navegador. Así si se interrumpe la carga por alguna razón, los datos quedan guardados y no tiene que completarlos de nuevo.

Los datos de local storage se borran una vez que se envía el formulario, o pasados 30 minutos
