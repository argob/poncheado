Páginas de formularios

Los formularios están compuestos de páginas de carga de parte del usuario y de respuesta de parte del sitio web.

Páginas de carga de datos

La página o páginas de carga de datos son las que requieren que el usuario piense, en algunos casos busque y complete datos necesarios para usar un servicio. Existen diferentes variables según la necesidad y el servicio.

Consulta o formulario simple

Sirve para servicios con respuestas simples como consultar la vigencia y fecha de algún documento

  • Pedir no más de 10 datos.
  • La página de respuesta debe mostrar los datos de la consulta y un botón para hacer otra consulta.

Ejemplos:

  • Datos personales completos

Formulario largo

Sirve para servicios que necesitan muchos datos del usuario o de una situación. Idealmente se dividen en varias páginas.

  • Dividir el formulario en varias páginas, en donde cada página tiene un grupo de datos específico.
  • Incluír una página de confirmación de datos que permita verlos todos juntos con links para editar cada página.

Páginas de respuesta

La página o páginas de respuesta son las que le aparecen al usuario luego de que envíen el formulario. Existen diferentes variables según la necesidad y el servicio.

Respuesta positiva

Sirve para cuando un formulario:

  • Se envía con éxito.
  • Se realiza un registro (alta, baja o modificación).
  • Se encuentra un documento vigente.

Respuesta negativa

Sirve para cuando un formulario:

  • No se envía por un error en el servicio (incluye un mensaje aclarando si el envío se va a hacer automáticamente después o no).
  • La acción que quiere hacer el usuario no se puede hacer por algún dato cargado (que no puede validar errores en la pantalla de carga por tener que consultar con una base de datos para responder).
  • Se encuentra un documento vencido.

Revisar correo electrónico

Sirve para casos específicos donde inmediatamente luego de la carga del formulario el usuario debe revisar su mail para continuar un proceso. Por ejemplo el registro para una plataforma que requiere que el usuario valide su cuenta de correo electrónico antes de poder usarla.

Comprobante de envío

Sirve para casos específicos donde el usuario va a continuar un proceso días más adelante. Por ejemplo cuando se hace un reclamo o se pide un turno.

Correos electrónicos

Sirven para enviar al usuario información y acciones que necesita completar luego de enviar un formulario, o para enviar comprobantes (por ejemplo de turnos o reclamos).
